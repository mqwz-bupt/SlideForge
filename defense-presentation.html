Background
选题背景与项目目标
解决什么问题？核心目标是什么？
问题痛点
模板千篇一律 — WPS/Canva 缺乏个性
内容重组困难 — 论文到 PPT 大量复制粘贴
迭代修改耗时 — 调整布局需逐页手动修改
核心目标
1. 上传文档，AI 自动生成结构化大纲
2. 16 套差异化视觉风格，覆盖四大类别
3. 自然语言对话实时修改幻灯片
4. 一键导出 HTML / PPTX / PDF 三种格式
目标用户
中国大学生和高校教师 — 上传课程笔记或论文，选择风格，AI 生成完整 PPT，对话微调后导出分享

Workflow
核心使用流程
六步从文档到精美演示文稿
1
上传文档 — PDF / DOCX / TXT / MD 文件解析为纯文本
2
AI 生成大纲 — 结构化章节+要点，可编辑排序
3
选择风格 — 情绪标签推荐 + 16 套模板浏览
4
AI 生成幻灯片 — 自动选择最佳布局
5
AI 对话微调 — 自然语言修改，实时预览
6
导出分享 — HTML / PPTX / PDF 一键输出
Prompt 工程设计
上下文注入 — 全部幻灯片 JSON 注入 System Prompt

格式约束 — ```slide-updates``` 代码块返回结构化 JSON

内容规范 — body 每条 ≤25 字，禁止 Markdown

清洗管道 — 自动剥离加粗/标题/列表符号，布局容错映射

Highlights
技术亮点
核心难题的解决方案
多模型适配器
统一 OpenAI 兼容接口，baseURL + apiKey 切换 GPT-4 / Claude / DeepSeek / 通义千问，业务代码零改动
双轨样式系统
主进程 StyleConfig（导出）+ 渲染进程 DesignTokens（预览），确保"所见即所出"
Prompt 工程 + JSON 容错
上下文注入 + 格式约束 + 清洗管道，支持两种代码块格式和布局容错映射
零依赖 HTML 导出
CSS/JS/字体 URL 全部内联为单文件，clamp() 响应式适配任意屏幕
流式输出 + 实时预览
IPC 流式传输逐字显示，Zustand 响应式驱动 React 组件重渲染
安全密钥管理
Electron safeStore API 加密存储 API Key，IPC 仅暴露必要接口
